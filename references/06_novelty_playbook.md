# 新颖点提炼 Playbook（用于 LLM / 人工复核）

> 目标：把“特征×对比文件”结果，变成可用的新颖点（差异特征组合）与风险提示。

## 1) 先选“最接近对比文件”
通常选 1–3 篇最接近的对比文件：
- 优先：与本发明问题/场景一致、且矩阵整体匹配度高的专利
- 其次：标题/摘要相近但 claims 匹配度一般的专利（可能术语不同）

## 2) 做特征映射（F → claims）
对每篇最接近对比文件：
- 把每条 F 标注为 YES/PARTIAL/NO
- YES/PARTIAL：引用 claims 命中片段
- NO：标注“未在抓取到的 claims 文本中命中关键术语（可能存在术语差异，需人工阅读全文确认）”

## 3) 候选新颖点写法：差异特征组合
新颖点尽量写成“组合”，而不是孤立单特征：
- NP1 = F2 + F5 + F7（组合后形成完整技术方案闭环）
- 每个 NP 必须解释：
  - 技术作用：为什么这些差异能解决技术问题
  - 互相配合：为何组合是不可替代/非显而易见（初步解释）

## 4) 风险点与补强建议
典型风险：
- 术语差异：对比文件可能用不同表达覆盖了你的特征
- 抓取失败：claims 解析为空/残缺
- 摘要不充分：摘要没写但 claims 写了

补强建议：
- 扩展检索式（同义词/英文/IPC）
- 把 Top3 的独立权利要求人工逐句比对
- 对关键效果补实验或补对比配置说明

## 5) 输出建议格式（LLM）
- 最接近对比文件列表（3条内）
- 每条候选新颖点：NP#、涉及 F#、最接近对比文件、差异描述、风险/建议
