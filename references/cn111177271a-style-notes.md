# CN111177271A 说明书写作思路（提炼）

1. 结构顺序固定：技术领域 -> 背景技术 -> 发明内容 -> 附图说明 -> 具体实施方式。
2. 段落编号连续：全篇使用 `[0001]` 递增编号，不在章节间重置。
3. 技术领域极简：一段一句，采用“本发明涉及...尤其涉及...”句式，直接点明主题和类型。
4. 背景技术分层写：先列举现有方案，再逐项指出资源占用、灵活性、兼容性等缺陷，结尾收敛到待解决问题。
5. 发明内容采用“目的 + 方面”推进：
   - 先写“本发明的目的是...”；
   - 再写“为实现上述目的，根据本发明实施例的一个方面...”给出方法；
   - 连续使用“进一步地”展开可选特征；
   - 再给出“另一个方面”的装置/系统、计算机设备、存储介质。
6. 有益效果单独成段：放在发明内容末尾，要求与背景问题一一对应，不用宣传语。
7. 附图说明有引导句：先写“附图仅用于示出实施方式，不构成限制”，再逐图说明图名和用途。
8. 具体实施方式先免责声明：先写“仅用于解释，不用于限定”，再进入实施例一/二/三。
9. 实施例展开方式：按“步骤/模块/单元”组织，每个实施例都回扣前述技术问题与发明内容。
10. 保护范围兜底句：末尾给出“在不偏离构思前提下的等效替换落入保护范围”。
11. 摘要句式遵循“本发明提供了…其中，该方法应用于…包括：步骤1；步骤2；步骤3。通过本发明，可…同时…。”，一般为单段短文并收束到2-4个效果点。
